GOS

Go Out in the Streets

*Alert:

This is just a backend application. Test using tools like Postman or any REST API client.

Project Description:

  1. GOS is an online service that helps people to find unfamiliar but yet interesting places and venues within the city.
  2. This will help people to save time, because they don’t need to browse the web or go through different apps and websites to find and decide the place they really want to go.
  3. GOS helps interesting small places built with innovation to reach a large number of people.
  4. GOS authenticates users with token based authentication using JWT (JSON Web Token)
  5. The service manages the outlets database and users database in MongoDb (MongoDb Atlas), a noSQL database to provide a seamless experience for all users.
  6. REST API Specification:
    • The REST API of GOS is made possible with the help of Express
    • GOS is supported by 3 routes “/outlets” ,"/favorites" and “/users”.
    • The “/outlets” route supports the GET request and POST request only for the reviews
    • POST request for outlets is restricted only to the admin.
    • There is sub route “/outlets/:outletID” to handle GET request for specific outlets details.
    • The “/users” route manages the user signup, login, and logout
